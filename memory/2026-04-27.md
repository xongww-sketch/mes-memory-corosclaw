## Q&A 全量回答记录（12个问题）

### Q1: 样机领走后工单计划数要不要减？
**回答**：计划数不变，完成数=良品+样机=planQty才算结单。**熊旺同意**。

### Q2: 样机在外观检之前还是之后拿走？
**回答**：都有可能。研发阶段随时可能，量产阶段也不固定。

### Q3: 看板是大屏还是PC网页？
**回答**：**一期就得做大屏**（65寸）。

### Q4: 成品料号前缀确认
**回答**：**一期确认只有HS开头**（不是上周说的BS/HS/JS/WQ四种）。只有HS料号可以自动关单。

### Q5: CREATED状态能否修改计划数/删除工单？
**回答**：**永远支持修改计划数量**（不限状态），删除工单目前没有这个操作。

### Q6: 关单后能否重新打开？
**回答**：一期不能，二期再做反结单。**熊旺认同**。

### Q7: 补料SN自动生成还是人工确认？
**回答**：报废后待补料+1，**去现有SN生成页面直接生成**。在现有Lowcode"底壳号生成工具"上加一个"补料生成"Tab。工单下拉只显示有待补料的。生成完后待补料-1。
- 熊旺强调：**以后所有界面都可能迁到MES系统，现在简单做，怎么方便怎么来**

### Q8: 补料SN在看板上怎么体现？
**回答**：补料SN的工序进度计入原工单的四工序进度，和普通SN一样走流程。**熊旺同意**。

### Q9: 人工关单时要不要记录差异数量？
**回答**：可以加 closeReason + closeGap。

### Q10: 任意工序都能转入维修吗？
**回答**：**是的，任意工序都能走入维修**。而且维修和报废**先做到一个页面**。

### Q11: 异常SN阈值？
**回答**：**48小时**。

### Q12: 发现异常SN后的操作流程？
**回答**：异常SN → 找出来 → 去生成SN的码（重新打码）。
**报废只存在于中壳报废场景**：上面镭雕了SN，所以会解除绑定大部分组件（主板、电池等）。
- 注意：这和之前理解的不同，报废不是简单的SN状态变更，而是涉及组件解绑

## 一期方案收敛结果

### 工单状态流转（确认版）
```
CREATED（已创建）→ PRODUCING（生产中）→ CLOSED（已关单）
```
- CREATED → PRODUCING：第一个SN扫码进站时触发
- PRODUCING → CLOSED：自动关单（HS料号）或手动关单
- 任何状态都可以修改计划数量
- 关单后不能重新打开（一期）

### 自动关单规则（一期确认版）
- **仅 HS 开头料号**可自动关单
- 条件：completionQty >= planQty 且 repairingQty = 0 且 pendingReplenishQty = 0
- completionQty = saleableGoodQty + sampleQty

### 看板设计（一期大屏版）
**顶部统计卡**（5个）：活跃工单数 | 今日计划工单数 | 总计划数 | 总完成数 | 总在制数
**主表列**：工单号 | 工单类型 | 计划数 | 销售良品 | 样机占计划 | 完成数 | 维修中 | 报废 | 待补料 | 剩余待完成 | 当前主阶段 | 四工序进度
- 四工序表头要加工序短名：老化/点胶/静置/外观
- 当前主阶段显示工序短名
**底部汇总**（4个）：总维修中 | 总报废 | 总待补料 | 可结单工单数
**右侧面板**：异常工单聚焦

**活跃工单池条件**：status=PRODUCING，未关单不下看板
**异常工单条件（一期简化）**：repairingQty > 0 或 pendingReplenishQty > 0

### SN查询页（3个Tab）
放到通用查询菜单下：
1. **工单SN列表**：输入工单号 → 展示所有SN + 当前工序 + 状态 + 最后扫码时间
2. **无动态SN**：从未扫码的SN → 导出补码
3. **异常SN**：有记录但48小时无后续动作 + 非完成/报废/维修中 → 产线排查

### 维修/报废（改现有页面）
- 基于现有Lowcode"维修记录"页面改
- **新增"维修结果"字段**：修复 / 报废
- 选"报废"→ SN状态变SCRAPPED → scrapQty+1 → pendingReplenishQty+1
- 报废时显示解绑选项（主板/电池/其他）
- 一期不做报废审批，直接确认

### 补料生成（改现有页面）
- 基于现有Lowcode"底壳号生成工具"改
- 加"补料生成"Tab
- 工单下拉只显示 pendingReplenishQty > 0 的工单
- 选工单后自动带出产品信息
- 生成后：pendingReplenishQty -= N, replenishQty += N

### 样机处理（待确认方案）
提出了"关单时算总账"方案：
- 不要求实时登记
- 手动关单时弹窗，必须填样机数量
- 差异=计划-良品-报废-样机，差异>0要填说明
- 有样机被拿走 → saleableGoodQty < planQty → 自动关单条件不满足 → 变成手动关单 → 必须填样机数 → 闭环
- **Q13 待熊旺确认**：这个方案能接受吗？
