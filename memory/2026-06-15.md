## 🔧 activation_records 表治本改造

**发起人**：熊旺（品质团队需求驱动）
**目标**：让品质团队能追溯到每台手表的**原始激活日期**，计算「从第一次激活 → 登记不良」的间隔，推算使用寿命。

### 业务问题

品质团队发现当前 `activation_records` 表存在一个致命缺陷：手表换主板重新激活时，旧激活记录被删除、新记录插入，**原始激活时间永久丢失**。这导致品质无法准确分析产品使用寿命。

此外，熊旺手上有 25 年/26 年国内/海外 4 个 Excel 汇总文件，包含大量历史首次激活时间数据，需要导入系统。

### 解决方案（已确认执行）

**核心设计**：
1. **停止删除旧激活记录** — 改为标记 `deactivestatus='0'`（失效），保留历史记录
2. **新建 `sn_first_activation` 表** — 独立存储每个 retroid 的第一次激活记录（不可变，source=excel/automatic）
3. **历史数据导入** — 4 个 Excel 文件去重后批量导入 `sn_first_activation`
4. **品质查询走新表** — Defective 系列 API 查询原始激活时间时，从 `sn_first_activation` 表获取
5. **判重规则** — 同日重复激活视为脏数据清理

**数据验证结果**：
- `sn_first_activation`（source='excel'）：25,448 条
- 与 activation_records 匹配：25,423 条（25 条不匹配，可能 retroid 已变更）
