## MES 接口分类讨论

### 熊旺提问
"所有接口分为哪几种类型，怎么区分，哪些适合写在n8n，哪些适合写在后端代码里"

### 结论：四类接口分类标准

| 类型 | 写在哪 | 判断标准 |
|------|--------|---------|
| A类：产线实时接口 | n8n | 产线设备（扫码枪/上位机/小盒子）直接调用 |
| B类：后台管理接口 | NestJS后端 | MES后台页面调用，需权限/事务/审计 |
| C类：跨系统集成 | 看对接方式 | REST API→n8n，SDK/MQTT→NestJS |
| D类：定时/批处理 | 看复杂度 | 简单→n8n，大数据量/队列→NestJS |

### F3/F4/F5接口重新分类
- **n8n（3个）**：repair_checkin_api（F3进站）、repair_message改造、repair_message_line改造
- **NestJS后端（6个）**：sn_query_api、sn_void_precheck_api、sn_void_api、sn_export_api、replenish_precheck_api、sn_replenish_api

### 核心原则
> 产线设备调的 → n8n | MES后台页面调的 → NestJS后端

## cron定时任务优化

### 问题
cron任务（isolated session）调飞书API拉聊天记录，因无用户授权导致失败，记忆文件变成空壳。

### 解决方案
- **改前**：cron调飞书API拉聊天记录 → 经常失败
- **改后**：main session对话中实时写记忆 → cron只负责兜底检查+推GitHub
- cron payload已更新，不再调飞书API
- 分工：记忆整理=对话中实时做，GitHub推送=cron 21:00兜底
