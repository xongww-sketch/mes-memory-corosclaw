## 📅 Day 1 — 2026-04-20（周一）业务方向收口 + 3份PRD生成

### 背景
熊旺集中讨论组装段需求，形成统一业务方案方向。

### 8个核心收口方向

#### 1. 工单结单方式调整
- 从"按SN维度结单"改为"按工单维度结单"
- 工单必须与计划数量对齐，报废不能直接冲抵计划
- 自动结单，不增加人工"待结单"动作
- 主状态：未开始 → 生产中 → 已结单
- 自动结单条件：
  - goodQty = planQty
  - repairingQty = 0
  - pendingReplenishQty = 0
  - wipQty = 0

#### 2. 补料机制
- 报废/损耗后必须补齐计划数
- 补料SN归属原工单，不开独立补料工单
- 先走"人工确认生成补料SN"轻量方案（不依赖审批流）
- 报废 = 系统可感知损耗
- SN物理丢失 = 系统不可自动识别 → 人工损耗登记
- pendingReplenishQty 作为待补料数量

#### 3. 维修/报废工序设计
- 维修工序：独立中间态，扫码进维修 / 扫码出维修
- 报废工序：独立终态，扫码确认报废
- 必须保留SN级事件记录，工单层做汇总统计
- 建议"SN级事件记录 + 少量业务标志 + 工单统计表"

#### 4. 组装看板
- 显示对象是**活跃工单池**（不是单纯"当天计划工单"）
- 活跃工单池包括：
  - 今日计划工单
  - 历史未结单工单（遗留）
  - 临时插单且已投产工单
  - 当前存在在制/维修中/待补料等状态的工单
- **核心原则：未结单不能下看板**
- 看板核心字段：工单号、分类、状态、计划数、良品数、维修中、报废、待补料、在制、剩余、当前主阶段、四工序进度
- 四工序必须体现"阶段到达数量"（不是累计扫码次数）
- 四工序：上架老化 → A/B壳点胶 → 功耗静置 → 外观产出
- 展示方式：四工序到达SN去重数量（如 100 / 92 / 80 / 75）

#### 5. 数量模型收口
- planQty / goodQty / repairingQty / scrapQty / replenishQty / pendingReplenishQty / wipQty
- remainQty = planQty - goodQty
- wipQty = actualInputQty - goodQty - scrapQty - repairingQty
- actualInputQty = 初始SN数 + replenishQty

#### 6. 数据模型建议
- 工单主表 + 工单统计扩展表 + 维修/报废/补料事件记录表
- 保持主表干净、利于看板查询、后续扩展容易

#### 7. 历史工单问题（重点）
- 历史未结单原因：SN丢码/报废后产线新开工单补量，系统两工单无关
- 结论：新规则上线后不再用"新工单补原工单"
- 历史数据需要**历史工单补料挂账修复方案**
- 允许人工补录"原工单-补料来源工单"关系

#### 8. 工单类型治理（重点）
- 当前混乱：组装/包装/维修/售后/反投 混用
- 收敛为：组装 / 包装 / 返投
- 维修工单类型仅保留"独立维修业务"
- 售后并入返投，试产需补充定义
- 补料不应再作为独立工单类型

### 一、销售预测与订单模式

#### 计划驱动链路
```
销售预测（12个月覆盖）
  → 3-4个月需求以GSG预测订单下到系统
    → 驱动MRP / 未来需求计划
      → 物料备料（提前2-3个月）
        → 实际工单下达（最多提前3天）
```

#### 关键认知
- **高驰是按库存生产**，不是OEM按订单生产
- 整机完成后**自动冲销预测订单**
- 预测订单不是真正的订单，只是数据/计划
- 备料与生产不同步：备料提前2-3个月，工单最多提前3天

```
┌─────────────────────────────────┐
```

#### 工单类型展示
- **组装**：正常生产，必须显示
- **试产**：必须显示
- **返工**：拆机返工显示，测试返工不显示
- **售后**：**不在组装看板显示**（在售后组单独显示）

#### 数量概念确认
| 字段 | 定义 | 说明 |
|------|------|------|
| 计划数 | SAP下达的订单数量 | 与SAP对齐 |
| 销售良品 | 产线外观检OK的数量 | 可用于销售 |
| 样机占计划 | 内部使用、不用于销售 | 从计划中占用 |
| 完成数 | 销售良品 + 样机占计划 | |
| 维修中 | 中间状态 | 最终会转为良品或样机 |
| 报废 | SN级报废 | 报废=待补料，一一对应 |
| 待补料 | 报废后需补SN的数量 | 与报废1:1对应 |
| 剩余待完成 | 计划数 - 完成数 | |

#### 关键公式
```
完成数 = 销售良品 + 样机占计划
待补料 = 报废数（一一对应）
剩余待完成 = 计划数 - 完成数
```

### 四、工单结单规则（工厂确认后重大调整）

#### 自动关单 vs 手动关单
| 类型 | 关单方式 | 判断依据 |
|------|---------|---------|
| BS/HS/JS/WQ开头的成品工单 | **自动关单** | 料号前缀判断 |
| 试产工单 | **手动关单** | 特殊情况多 |
| 售后工单 | **手动关单** | 随机性大 |
| 返工工单 | **手动关单** | 类型多、数量不确定 |
| 其他半成品/零部件 | **手动关单** | 需要人工确认 |

#### 结单与SAP的关系
- **MES关单 ≠ SAP关单** → 各自独立，互不影响
- MES显示两个状态：MES关单状态 + SAP关单状态
- **不能因为SAP关单了MES就自动关单**
- SAP关单后状态同步到MES，仅作展示参考

#### 结单条件
- 自动关单料号（BS/HS/JS/WQ）：completionQty = planQty + repairingQty=0 + pendingReplenishQty=0 + wipQty=0
- 手动关单：人工确认后操作

#### 外观检（FQC）作为结单依据
- 所有工单最终以**外观检OK数量**作为完成判断
- FQC只校验工序是否完成，不校验其他工序
- 样机领走不经过外观检，通过样机审批流程处理

### 五、返工工单处理（详细讨论）

#### 返工工单显示规则
| 返工类型 | 是否显示在组装看板 | 说明 |
|---------|-------------------|------|
| 拆机返工 | ✅ 显示 | 有独立SAP返工工单 |
| 测试返工 | ❌ 不显示 | 用原工单，不新建返工单 |

#### 返工工单拆分
- SAP一个返工工单可能包含多颗料（不同颜色/型号）
- **MES需要拆分**：一颗料对应一个MES工单
- 虚拟工单号 = SAP工单号 + 3位流水号

#### 返工工单数量联动
- SN从正常组装工单跳到返工工单时：
  - **原工单计划数 - 1**
  - **返工工单计划数 + 1**

#### 返工SN处理
- 返工时可能需要重新生成SN（如变色）
- 统一规则：所有SN统一生成新SN
- 镭雕SN不变的情况：保留原SN
- 扫入返工工单的数量不能超过返工工单计划数

### 六、样机处理

#### 样机领走规则
- **任何工单类型**（组装/试产/返工/售后）的任何阶段都可以领走
- 样机领走需要**审批流程**（类似飞书审批）
- 审批通过后：样机占计划 +1，完成数 -1
- 样机不经过外观检，不受工序限制

## 🔑 关键业务模型（终版收口）

### 工单状态流转
```
未开始 → 生产中（有SN投入） → 已结单
```

### 核心数量公式
```
完成数 = 销售良品 + 样机占计划
待补料 = 报废数（1:1对应）
剩余待完成 = 计划数 - 完成数
completionQty = saleableGoodQty + recognizedSampleQty
wipQty = actualInputQty - goodQty - scrapQty - repairingQty
actualInputQty = 初始SN数 + replenishQty
remainQty = planQty - goodQty
```

### 结单条件
**自动关单**（仅BS/HS/JS/WQ开头料号）：
```
completionQty = planQty
repairingQty = 0
pendingReplenishQty = 0
wipQty = 0
```

**手动关单**（试产/售后/返工/半成品/零部件）：人工确认后操作

### 自动关单 vs 手动关单
- 自动关单：BS/HS/JS/WQ开头的成品工单
- 手动关单：试产、售后、返工、半成品/零部件
- MES关单 ≠ SAP关单 → 各自独立，双状态展示

### 四工序
```
上架老化(A) → A/B壳点胶(B) → 功耗静置(C) → 外观产出(D)
```

### 看板展示
```
顶部统计卡：活跃工单 | 计划 | 完成 | 维修 | 报废 | 待补料 | 异常工单 | 可结单工单
主表：工单号 | 类型 | 状态 | 计划 | 完成 | 维修 | 报废 | 待补料 | 在制 | 当前阶段 | 四工序进度
```

### 看板显示规则
- 组装：显示
- 试产：显示
- 返工：拆机返工显示，测试返工不显示
- 售后：不在组装看板显示
