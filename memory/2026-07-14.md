### 维修场景分析（今天深度讨论，仍未闭环）

**核心铁律**：维修后能不能激活，只看主板芯片里有没有一个「没被用过 + 已在 Apple 注册」的 provisioning token。

**维修分水岭**（从 Apple 能激活的角度反推）：
- 旧板**从没激活**，拆到新 SN → 三条件满足，直接能激活，只需改 MES 绑定关系
- 旧板**已激活**，拆到新 SN → 必须先解绑，否则 UUID 在 Apple 侧还占着 → 激活失败
- 换新板 → 必须发新 token，但前提是主板能被 Apple 注册

**卡住的两个死结**：
1. **MES 不能单方面解绑已绑账户的主板** — R3 §6.4 白纸黑字：unpair 必须由当前配对的 owner device 通过蓝牙发起，没有任何服务器 API 让第三方绕过。这是 Apple 防盗设计。→ **已激活的主板，MES 侧救不了，只能原用户在 App 里移除设备**

## 📌 二、越南工厂逻辑（=日本工厂逻辑）

**背景**：信息化业务人告诉熊旺，参考两份越南业务文档，说这就是日本工厂的逻辑。

### 核心定调：越南工厂不是高驰的，是佳禾的场地和人

所以本质是**高驰不自己造，而是「买佳禾的成品来卖」**，中间用一串采购/销售单据把货权和账务串起来。日本工厂如果是同样的代工逻辑，就照这套搬。

### 三方角色

| 主体 | 角色 |
|---|---|
| 广东高驰 | 出原材料（卖原料给广东佳禾） |
| 香港高驰 | 中间商+账务主体（采购成品、卖给最终客户） |
| 佳禾（广东/香港/越南） | 代工方，越南工厂实际造货 |

### 完整业务链

```
① 销售预测 → 香港高驰在 OMS 对「香港佳禾」下【成品采购订单】
② 计划部定生产计划 → 通知佳禾下原料采购单
③ 广东高驰 → 卖原材料给广东佳禾（OMS 销售订单+交货单，原料实物发出）
④ 越南工厂收到原料 → MES 下生产工单开造
⑤ 越南 MES 产成品 → 扫码入库到「越南仓」（OMS 越南 SN 库存表）
⑥ 香港高驰出货平台扫码出库 → OMS 扣库存
⑦ 香港高驰卖成品给最终客户（OMS 销售订单→交货单→过账）
```

### 为什么非得这么绕？（三大驱动力）

1. **税务合规**：越南是外国，不能内部调拨，必须走「买+卖」的商业交易关系；越南出口有退税/关税政策，用贸易关系才能合规享受
2. **货权清晰**：每一步货权转移都有单据（交货单）作证，谁持有货、谁欠钱一目了然
3. **账务可对**：香港高驰是独立核算主体，通过中间商身份，国内采购成本和海外销售收入的差就是利润，财务链路完整

**操作在越南工厂，但账务全挂香港高驰。** 关键点是**「货权转移必须有单据链」，不能靠内部调拨**。

### 与日本工厂的关联

日本工厂如果是同样的代工模式（工厂产权不是高驰的），逻辑完全一致：
- 高驰通过商业交易（采购/销售）利用日本产能
- 不在日本建 SAP 公司代码
- 账务挂香港或国内主体
