## 测试环境数据仿真方案（凌晨 02:24-03:22）

### 需求背景
熊旺提出：线上环境 SN 绑定数据正常且实时，但测试环境永远没有数据。需要一套方案，根据测试环境的工单和生成的 SN，自动生成高仿真的绑定数据。

### 核心约束
- **完全独立**：测试环境有独立的 DB 和 n8n，与生产环境物理隔离
- **高仿真**：字段格式、时序、渠道映射都要贴近真实数据
- **数据范围**：工序绑定记录（SN 过每道工序）
- **触发方式**：由虾王推荐（最终采用基于工单 retroid 批量插入）
- **数据量**：全 18 条都造（一个工单下多个 retroid，每个 retroid 18 条 machine_data 记录）
- **daily_info 表不用聚合**，只需要绑定数据

### 技术方案
- **目标表**：`mes.machine_data`（测试库 mes，151 机子）
- **插入逻辑**：通过 `joborder_id` 查出多个 retroid → 为每个 retroid 插入 18 条 machine_data 记录
- **machine_data_id**：自增（不要手动指定）
- **参考数据**：熊旺提供了一条完整 SN 绑定数据的 SQL INSERT 语句作为模板

### 遇到的问题
- **duplicate key 错误**：`Key (machine_data_id)=(0) already exists`
- **根因**：machine_data_id 没有设置自增，插入时传了 0 导致冲突
- **解决方向**：设置表自增（ALTER SEQUENCE 或修改列定义）
