## 售后维修与 MES 边界讨论

**熊旺提问**：MES 应该承接售后维修重新绑定的子数据吗？

**分析结论**：MES **不应该**主动承接售后维修的子数据。

**核心理由**：
1. **职责边界**：MES 是制造执行系统，管产线制造段（SN生成 → 工序绑定 → 成品出站）；售后是成品出厂后的生命周期，应由售后系统（如售后宝）管理
2. **场景拆分**：
   - 售后换中框需重新过 MFI 绑定 → 可能需要走 MES 工序
   - 售后只换配件不涉及 SN/token → 售后系统自己闭环
   - 售后修改设备 ID 推送 → 不在 MES（此前已验证 322 个工作流不存在此链路）

**待确认**：熊旺尚未回复最终决策，需确认售后维修回厂后是否走产线工序。
