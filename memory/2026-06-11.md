## 🚀 n8n-mcp-gateway 项目（今日核心交付）

### 背景
熊旺要求搭建团队级 n8n 只读查询 MCP 网关，部署在 192.168.64.54，让团队内任意 agent（Claude Code/Cursor/Codex/OpenClaw 等）都能通过标准 MCP 协议查询 n8n 工作流信息，无需暴露 n8n 真实 API key。

### v0.1 — 基础网关搭建
- **代码**：workspace/n8n-mcp-gateway/（Node+TS+@modelcontextprotocol/sdk，Streamable HTTP）
- **部署**：/opt/n8n-mcp-gateway/，docker compose，容器名 n8n-mcp-gateway
- **端口**：192.168.64.54:9130（绑内网网卡，非 0.0.0.0；服务器仅私网 IP 无公网，物理隔绝外部）
- **架构**：token → 角色 → 能力白名单 → n8n key 池，四层解耦（config/tokens.json，不进 git）
- **4 个只读工具**：find_workflow / find_email_recipients / read_workflow_logic / find_failed_executions
- **验证通过**：healthz ok / 无 token=401 / 带 token initialize 握手成功 / 工具能力已声明
- **凭据**：全在 .secrets（不进 git）：server_64_54.env(SSH) / n8n_api.env(新只读 key) / gateway_tokens.env(master+team token)
- **设计文档**：https://www.feishu.cn/docx/Eh8Xd4kv0oHF1AxzzlJcBhV7nHf

### v0.4 — REST 查询端点
- 新增 /api/query/{find_workflow,find_email_recipients,read_workflow_logic,find_failed_executions}?参数
- Bearer token 鉴权，返回 JSON
- 目的：让"直接 curl"型 skill 可用，不依赖 MCP 客户端

### Skill 封装与分发
- **n8n-query skill**：封装"接入 n8n MCP"流程，agent 读了能自动按自己工具类型装上 MCP
- **两个 skill 上传到 COROS Skills Registry**（http://skills.apps.coros.team）：
  - **n8n-mcp**：教 agent 自动装 MCP（Claude Code/Cursor/Codex/OpenClaw）
  - **n8n-curl**：直接 curl REST 查（免 MCP，任何能跑 shell 的 agent）
- **最终统一方案**（参考 aihot 方式）：一个 SKILL.md，装上后用户自选装 MCP 或直接 curl
  - 网关加路由 /skill/:name 和 /skill/:name/SKILL.md，直接返回 text/markdown 原文
  - SKILL.md 挂载：/opt/n8n-mcp-gateway/skills/<name>/SKILL.md（compose volume :ro）
  - **统一安装 URL**：http://192.168.64.54:9130/skill/n8n
  - 一个 SKILL.md 含：二选一（A 接 MCP / B 直接 curl REST）+ 逐平台 MCP 配置写法 + 四接口 curl 示例
- 平台无真 zip 下载，所有端点返回 JSON（files[].content），clawhub 靠解析 content
