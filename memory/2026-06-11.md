## 🔧 Cron 定时任务修复（今日彻底解决根因）

### 真正根因（终于定位）
- 表象：每日记忆同步任务"反复丢失"，6/7~6/10 几天没跑
- **真因**：`jobs.json` 文件里原版任务一直好好存着（ID `a1dbb5ff`，带 51 秒成功记录），但**运行时调度器内存没加载它** → 文件与调度器不同步 → 任务永不触发
- 之前误判为"文件丢了"，用 systemEvent 重建了劣化版（少了提取脚本+GitHub推送流程），导致 5ms 空转假成功

### 排查中踩的坑（教训）
- `at` 一次性任务跑完会自动 `enabled:false` → 看起来像"消失"
- `deleteAfterRun:true` 跑完直接删 → 真消失
- systemEvent 触发跑在隔离 run session，拿不到主会话上下文，做不了"回顾今天对话"
- 正确做法：直接建 `cron` 循环 + `agentTurn` 完整流程，不用 at/不删/不重建

### 最终修复
- 删掉所有劣化版/测试任务，用原版完整 agentTurn 配置重新注册进调度器
