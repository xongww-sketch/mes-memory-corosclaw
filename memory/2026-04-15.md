## MES Server代码深度分析 (03:26 UTC)

熊旺直接发送了 `coros-mes-prod` 的 zip 文件（绕过 git 认证问题）。
项目已解压到 `/tmp/mes-server/coros-mes-prod/`。

### 技术栈
- **后端**: NestJS + Prisma + PostgreSQL + Redis + BullMQ
- **前端**: Vue3 + Vite (基于若依RuoYi-Vue3)
- **集成**: 飞书(Lark SDK) + MQTT + n8n + FTP + SSH
- **架构**: carole-admin 全栈快速开发平台

### 核心Prisma Schema (7个schema文件)
1. `system.prisma` — 系统管理 (用户/角色/菜单/部门/字典/日志)
2. `commonPlan.prisma` — 基类计划 (CommonPlan)
3. `productionPlan.prisma` — 生产计划 (ProductionPlan, 含工单/产线/排班)
4. `ticketLog.prisma` — 工单日志
5. `dataLog.prisma` — 数据审计日志 (自动记录所有写操作)
6. `deviceEnvironment.prisma` — 设备环境 (温湿度/CO2)
7. `introspected.prisma` — 内省模型 (Applemfi, Resources, Devices, Products, ProductModel, PrintTemplate, InstallationPackage等)

### 核心业务模块
- **SN管理**: 批量导入、SN编码规则(10位)、产品型号映射、底壳旧SN上传
- **工单(Ticket)**: 创建/批量创建/搜索/更新/删除，关联生产计划
- **生产计划(ProductionPlan)**: 与CommonPlan关联，含产线/排班/数量追踪
- **Dashboard**: 产线实时监控，工单状态统计
- **数据查询**: SN查询/FQC/上位机/包装/维修/温湿度
- **设备管理**: MQTT实时连接，设备环境监控
- **软件包管理**: 安装包上传/下载/版本管理
- **打印模板**: 标签打印模板CRUD

### 关键集成
- **飞书**: OAuth登录 + 审批流(缺陷退货) + WebSocket事件 + BullMQ队列
- **n8n**: HTTP调用n8n工作流，支持dev/test/staging环境自动加_test后缀
- **MQTT**: 设备实时通信
- **FTP**: 文件传输（安装包/打印模板等）
- **SSH**: 远程脚本执行

### 认证架构
- JWT Token + Redis缓存
- 飞书OAuth登录（通过email匹配系统用户）
- RBAC权限: 角色→菜单→权限字符串，`*:*:*`为超级管理员
- IP黑名单、验证码、接口限流(ThrottlerModule)

### PrismaService亮点
- 自动审计: 所有写操作自动记录DataLog（通过Prisma Extension）
- 使用RequestContext获取操作者ID和请求ID
- 跳过审计的模型: DataLog, Resources, ProductionPlan, ticketLog, SysLogininfor
