## 🚀 改造方案（完全基于 n8n + Apple SWA API）

根据 Apple R3 规范和分析，这是我的推荐方案：

### 你需要新建/改造的工作流

#### **一、Token 池管理（全新）**

**工作流 A1：`apple_fetch_token_batch`**（定时拉取）
- 定时触发（可配置，如每周/按需）
- POST `https://swa.apple.com/api/v1.0/external/authEntityRequests`
  - Body: `{"ppid": "xxx", "requested_auth_entity_count": 50}`
- 保存 `request_id` 到 `apple_token_batch` 表
- N 分钟后拉取结果（Apple 异步生成）
- GET `https://swa.apple.com/api/v1.0/external/authEntities/{request_id}`
- 下载 CSV → 解析 → **唯一性校验** → **格式校验（validate_auth_entity）** → 写入 applemfi

```sql
-- 新增 apple_token_batch 表
CREATE TABLE mes.apple_token_batch (
  id SERIAL PRIMARY KEY,
  ppid VARCHAR(50) NOT NULL,
  request_id VARCHAR(100) NOT NULL,
  count_requested INT DEFAULT 50,
  count_received INT DEFAULT 0,
  status VARCHAR(20) DEFAULT 'PENDING', -- PENDING/SUCCESS/FAILED
  create_time TIMESTAMP DEFAULT NOW(),
  complete_time TIMESTAMP
);
```

#### **二、Token 唯一性 + 合法性校验引擎（全新 Code 节点）**

这是**核心**。在下载 CSV 解析后、写入 applemfi 前执行：

**校验规则（基于 Apple R3 规范 3.4/3.5）：**

| 校验项 | 检查内容 | 拒绝条件 |
|--------|---------|---------|
| Token 长度 | Base64-encoded Token | 非空且长度达标 |
| UUID 格式 | 30 hex chars | 非 hex / 长度≠30 |
| Token 去重 | 查 applemfi 已有 `Base64-encoded Token` | 重复则丢弃/告警 |
| CRC16 校验 | token 的 CRC16 CCITT | 重新算 CRC 对比不一致 |
| ppid 匹配 | 该行的 ppid 与请求一致 | 不一致告警 |

**关键 - Base64-encoded Token 本身即 Apple 下发**，其内部结构和长度在 Apple 规范有明确定义（ECIES 加密结构），可以通过解析其前导字节做基本合法性校验。

#### **三、产线上料 API（改造 MFI写入工序）**

**`/MFI_procedure`**（POST，产线调用）
1. 校验 SN 格式（10位长度）
2. 查 `cplc` 表确认前工序完成
3. **查 applemfi 表取未绑 token（retroid IS NULL，优先级：无 UUID > 有 UUID）**
4. **NEW：加 TRY-LOCK 机制**——用 UPDATE ... WHERE retroid IS NULL LIMIT 1 RETURNING * 防止并发争抢同一 token（当前代码是 SELECT 再 UPDATE，有 race condition）
5. 生成 UUID（Crypto 节点）
6. **NEW：Token 存量预警检查**——若剩余 < 阈值（如 20），触发告警
7. CRC16 转换 uuid+crc / token+crc
8. 写入 uuid 到 applemfi 该行
9. 回给产线：`{code:200, uuid, token, uuidcrc, tokencrc}`

#### **四、写入结果回执 API（改造 MFI工序写入返回-新mes）**

**`/MFI_procedure-newmes_result`**（POST，产线写完固件后回调）
1. 校验入参（sn, uuid, token, uuidcrc, tokencrc）
2. 查 `applemfi` 匹配 uuid + (Base64-encoded)Token
3. 如果匹配到：
   - **NEW：调 Apple SWA 状态查询** `GET https://swa.apple.com/api/v1.0/external/status/{request_id}` 或 `GET https://swa.apple.com/api/v1.0/external/single/usedAuthEntity`（查 token 在 Apple 端是否已 used）
   - 仅当 Apple 端确认 → 写入 applemfi status=3, retroid=sn
4. 写入 machine_data（保持现有逻辑）
5. 若无匹配 → 报错

#### **五、SN 查 Token/UUID 接口（新建）**

**`/apple_get_sn_token`**（GET/POST）
- 入参：`sn`
- SQL：`SELECT retroid, token, uuid, "Base64-encoded Token", uuidcrc, tokencrc, status, completetime FROM mes.applemfi WHERE retroid = $1`
- 返回完整的 token 信息，供后续工序做比对校验

#### **六、Token 存量预警定时任务（新建）**

定时触发（如每 30 分钟）
- SQL：`SELECT ppid, COUNT(*) as total, COUNT(*) FILTER(WHERE retroid IS NULL) as available, COUNT(*) FILTER(WHERE retroid IS NOT NULL AND status < 3) as in_use, COUNT(*) FILTER(WHERE status = 3) as completed FROM mes.applemfi GROUP BY ppid`
- 若任一 ppid 的 `available < threshold`（如 < 20）→ 发飞书告警或邮件
- 同时检查是否有 `retroid IS NULL` 但 `uuid IS NOT NULL`（已分配未完成的脏数据）

#### **七、Token 状态回查 Apple API 接口（改造 Token注册状态查询）**

**`/apple_token_status`**（GET，Webhook）
- 入参：`request_id`
- 调 Apple SWA：`GET https://swa.apple.com/api/v1.0/external/status/{request_id}` 或 `GET https://swa.apple.com/api/v1.0/external/single/usedAuthEntity`
- 返回 Apple 端实际状态，确保不会出现「本地 status=3 但 Apple 端未使用」的不一致

## ⚠️ 关键注意事项

1. **Apple SWA API 认证**：当前所有 Apple 接口请求只带了 Accept/Content-Type header，没有看到任何 Bearer Token 或 Auth header，**这可能是工作流 inactive 的原因之一**。你需要确认 Apple 侧给 SWA API 的认证方式。

2. **并发抢 token**：当前 `MFI写入工序` 是先 SELECT 再 UPDATE，多条产线并发时可能抢同一个 token。**必须改为** `UPDATE applemfi SET retroid = $1 WHERE id = (SELECT id FROM applemfi WHERE retroid IS NULL AND ppid = $2 ORDER BY id LIMIT 1 FOR UPDATE SKIP LOCKED) RETURNING *`，或者用 Postgres 的 `UPDATE ... WHERE retroid IS NULL LIMIT 1`。

3. **状态机建议**：
   - `status=0`：已从 Apple 拉取但未分配
   - `status=1`：已分配给 SN 但产线未写入（有 uuid，但未收回报）
   - `status=2`：产线回报写入成功（Apple 端已 used）
   - `status=3`：已完全完成（写入 machine_data）

4. **UUID 生成策略**：当前用 n8n Crypto 节点生成 UUID，但规格里 UUID 需要是 30 位 hex（不是标准 UUID 格式），目前的 uuidGetSn 有各种 parsing 逻辑。需确认产线到底传什么格式。
