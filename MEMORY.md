### ⚠️ 一期重大校准（2026-04-27）

4/27 熊旺重新梳理了真实现状，推翻了部分上周设计。以下是**最新确认版**，以此为准：

#### 一期真实约束
- **没有SAP对接、没有WMS**
- **5月9号上线，2个开发**（约12个工作日）
- 已有：工单、SN生成规则（Lowcode）、维修工序页面（Lowcode）
- 一期明确不做：SN跨工单、样机审批、返工拆分、SAP状态、配置化筛选、历史挂账修复、报废审批流、缺陷代码库

#### 工单状态（确认版）
```
CREATED → PRODUCING → CLOSED
```
- 任何状态可修改计划数，不支持删除工单
- 关单后不能重新打开（一期）

#### 自动关单（确认版）
- **仅 HS 开头料号**（不是上周的4种前缀）
- 条件：completionQty >= planQty 且 repairingQty=0 且 pendingReplenishQty=0
- completionQty = saleableGoodQty + sampleQty

#### 组装看板（一期大屏版）
- 顶部5统计卡 + 活跃工单列表（12列含四工序进度）+ 底部4汇总 + 右侧异常面板
- 四工序表头加短名：老化/点胶/静置/外观
- 活跃工单池：status=PRODUCING
- 异常条件：repairingQty>0 或 pendingReplenishQty>0
- 筛选栏：工单类型(多选) + 工单状态(多选) + 物料号(搜索)
- 大屏适配：数字≥48px，表格内容≥24px

#### 维修/报废（方案重大演进 — 2026-05-07 更新）
**旧方案（F3）**：入维修/出维修/直接报废三个操作
**新方案（进站-出站模型）**：
- 进站：扫SN确认送维修 → status=4（维修中）→ repairingQty+1
- 出站三选一：
  - **修复**：填维修记录 → status恢复 → repairingQty-1 → 回产线
  - **报废**：填报废原因 → 解绑 → status=5 → repairingQty-1, scrapQty+1, pendingReplenishQty+1
  - **作废**：填作废原因 → status=3 → repairingQty-1
- 新增维修中SN列表Tab（显示滞留时长预警）
- 任意工序都能转入维修
- 工序扫码校验：status IN (1,2) 允许；status IN (0,3,4,5,6) 拒绝并给不同提示
- 一期不做报废审批

#### 补料（改现有SN生成页）
- 在"底壳号生成工具"上加"补料生成"Tab
- 工单下拉只显示有待补料的工单
- 生成后 pendingReplenishQty-=N, replenishQty+=N
- 补料SN归属原工单，和普通SN一样走工序

#### SN查询页（3个Tab）
- 工单SN列表 / 无动态SN导出 / 异常SN（48h阈值）
- 放到通用查询菜单下

#### 样机处理
- 方案："关单时算总账"——手动关单时必须填样机数量
- 有样机 → 自动关单条件不满足 → 变手动关单 → 填样机数 → 闭环
- **Q13 待熊旺确认**

#### SN 状态扩展（2026-05-07 确认）
- 原状态：0=反投机, 1=已生成, 2=已绑定苹果token, 3=已失效
- 新增状态：4=维修中, 5=报废, 6=丢失
- 作废复用 status=3（已失效）

#### 中框报废后SN处理
- **推荐方案**：闭环旧SN + 生成新SN（不复用旧SN）
- 行业标准：富士康/立讯/歌尔均为组件可复用、SN不复用
- 依据：ISO 13485 / IATF 16949 追溯性原则
- **待熊旺确认是否采纳**

#### 丢码处理方案
- 三层防线：预防（工序校验一进一出）→ 发现（异常SN 48h预警）→ 处置（标记丢失→缓冲期→报废）
- 丢失SN 24h缓冲期（可能找回来），status=6，找回可恢复，超时自动转报废
